Fortune 50 Company

Color Picker

  • My Role: UX Designer
  • Duration: 3 Months

Project Description

Design and implementation of a Color Family filter on Fortune 50 company retail site. Two separate Color Family filters were created; one for makeup products in eye, lipstick, blush, and nail polish categories, and another for hair dye.

My Role

The goal of implementation was to use an algorithm to sort products into the most accurate color families possible with the least amount of manual intervention. I was responsible for defining the color families and worked closely with the developers to make sure the algorithm ultimately yielded the best customer experience on the website. I also created wireframes for the filter design and interactions on the front end website.

The Challenge

Though many items have color swatches, no products belonged to a color family prior to this project. I outlined the following functional specifications as goals for the way the filter should work:

  • Selecting a color from the color picker should return only products of the selected color
  • Only colors relevant to the search results will appear in the picker (so no green in lipstick if we don’t have green lipstick)
  • When users select a color from the picker, the results page will update to only show the subset of products that match the selected color
  • The user can select multiple color families to view if they choose
Why is defining color complicated?

Color exists on a 2D plane, so defining a range is difficult. There are 16,777,216 possible colors using #RRGGBB notation. To illustrate this, take a look at the color picker below. Which coordinates in the image should define "red"?

Approach

Competitor Analysis

While the developers worked on extracting the hex values from color swatches, I looked at competitor sites to see how they were approaching their color pickers. Based on the hits and misses of these competitors, my takeaways were:

  • DO - make sure products are matched to color families as accurately as possible. One or two imperfect matches were forgivable, but too many mismatching made the filter seem broken.
  • DO - Make the granularity of color families and names relevant to the product categories. For example, make sure there are enough color families so the user can filter on "red" and "coral" lipsticks separately.
  • DO - label all color families with the color name.
Sorting Algorithm

After a few attempts, we ultimately created a two step system:

  1. To simplify the number of colors we were sorting, we first mapped the product swatches to the 148 defined web colors
    • "Web colors are colors used in displaying web pages, and the methods for describing and specifying those colors" (from Wikipedia). These are the colors you can write by name using CSS, such as "aliceblue" or "whitesmoke"
    • ~20 additional custom colors were used to improve matching accuracy. This was especially important for the Berry color family, which was not well represented by the web colors
  2. I then mapped the ~168 colors into the proposed color families
    • To do this, I looked through a spreadsheet output of the algorithm
    • This was ultimately 15 customer-facing color families for makeup, and 9 for hair color.

Testing

The color families were A/B tested by product prior to launch. Before launching the filters to all customers, they were tested with a few customers as part of a larger usability session by our user research team. The testing yielded no issues with the filter design.

Post-Launch

At the time of writing, the color family filters were the 3rd most used filter
